## Core Interfaces

| Interface | Responsibility |
|-----------|----------------|
| `AIProvider` | LLM chat, stream, embed |
| `MemoryProvider` | Conversations, user memory, sessions |
| `VectorProvider` | Embeddings storage, similarity search |
| `ToolProvider` | Tool listing and execution |
| `Agent` | Instruction-driven autonomous execution |

## AIRuntime

The `AIRuntime` class orchestrates all providers:

- `chat()` / `stream()` вҖ” LLM inference with optional memory persistence
- `embed()` вҖ” Generate embeddings via the active provider
- `remember()` / `searchMemory()` вҖ” User memory operations
- `retrieveContext()` вҖ” RAG context retrieval via vector search
- `runTool()` вҖ” Execute tools from registered providers

## Provider Switching

Set `AI_PROVIDER` in `.env`. The `fromEnv()` factory in `@kortex/bootstrap` dynamically loads the correct adapter вҖ” no code changes required.

## Data Flow (RAG)

1. **Ingest** вҖ” Documents chunked via `@kortex/rag`
2. **Embed** вҖ” Chunks embedded via active `AIProvider`
3. **Store** вҖ” Vectors stored in `VectorProvider` (pgvector)
4. **Retrieve** вҖ” Query embedded, similar chunks returned
5. **Generate** вҖ” Context injected into chat messages
